## USERS MODULE — /api/users

### GET /api/users
Description: Get paginated list of users
Query params:
  - page: number (default: 1)
  - limit: number (default: 10, max: 100)
  - sortBy: string (default: 'createdAt')
  - sortOrder: 'asc' | 'desc' (default: 'desc')
  - role: RoleCode ('USER' | 'ADMIN' | 'MANAGER')
  - search: string (searches firstName, lastName, email)
Response: PaginatedApiResponse<IUserSummary> (200)

### GET /api/users/:id
Description: Get single user by ID
Params: id (number)
Response: ApiResponse<IUser> (200)
Errors: 404 - Usuario no encontrado

### POST /api/users
Description: Create new user
Body: CreateUserDto { email, firstName, lastName, password, birthday?, role? }
Response: ApiResponse<IUser> (201)
Errors: 409 - El correo electrónico ya está en uso | 400 - validation errors

### PATCH /api/users/:id
Description: Update user profile (no password)
Params: id (number)
Body: UpdateUserDto { email?, firstName?, lastName?, birthday? }
Response: ApiResponse<IUser> (200)
Errors: 404, 409 (email conflict), 400

### PATCH /api/users/:id/toggle-status
Description: Toggle user active/inactive status
Params: id (number)
Body: none
Response: ApiResponse<IUser> (200) with updated active field
Errors: 404 - Usuario no encontrado

### PATCH /api/users/:id/password
Description: Change user password (requires current password verification)
Params: id (number)
Body: ChangePasswordDto { currentPassword, newPassword, confirmPassword }
Response: ApiResponse<null> (200)
Errors: 401 - La contraseña actual es incorrecta | 400 - Las contraseñas no coinciden

### DELETE /api/users/:id
Description: Permanently delete user
Params: id (number)
Body: none
Response: ApiResponse<null> (200)
Errors: 404 - Usuario no encontrado

### Notes:
- Password is never returned in any response
- Passwords are hashed with bcrypt (rounds: 10)
- birthday is an ISO 8601 date string (e.g. '1990-05-15')
- toggle-status is idempotent (can be called multiple times safely)

## AUTH MODULE — /api/auth
All routes under /api/auth. Login is @Public (no JWT required).
All other app routes require valid access_token cookie (global JwtAuthGuard).

### POST /api/auth/login                              [@Public]
Description: Authenticate user and issue JWT token pair as HTTP-only cookies
Body: LoginDto { email: string, password: string }
Response: ApiResponse<ITokenUser> (201)
Sets cookies:
  - access_token  (httpOnly, secure in prod, sameSite=strict, maxAge=15min)
  - refresh_token (httpOnly, secure in prod, sameSite=strict, maxAge=7d, path=/api/auth)
Errors:
  - 401 — 'Credenciales incorrectas' (email not found OR wrong password — same message, no enumeration)
  - 401 — 'Tu cuenta está desactivada. Contacta al administrador' (inactive user)
  - 400 — validation errors

### POST /api/auth/refresh                           [RefreshAuthGuard — reads refresh_token cookie]
Description: Rotate token pair. Issues new access + refresh tokens (refresh token rotation).
Old refresh token is invalidated in DB.
Body: none
Response: ApiResponse<ITokenUser> (200)
Sets cookies:
  - access_token  (new 15min token)
  - refresh_token (new 7d token — replaces old)
Errors:
  - 401 — 'Sesión expirada, inicia sesión nuevamente'

### POST /api/auth/logout                            [JwtAuthGuard — requires access_token cookie]
Description: Invalidate session. Clears refresh token from DB, clears both cookies.
Body: none
Response: ApiResponse<null> (200)
Clears cookies: access_token, refresh_token
Errors:
  - 401 — if no valid access_token cookie

### GET /api/auth/me                                 [JwtAuthGuard — requires access_token cookie]
Description: Get current authenticated user.
Used by Angular authGuard to hydrate state on page reload.
Response: ApiResponse<ITokenUser> (200)
Errors:
  - 401 — if no valid access_token cookie

### Security notes:
- Tokens stored in HTTP-only cookies (immune to XSS)
- Refresh token stored as bcrypt hash in DB (immune to DB dump)
- Refresh token rotation: each use invalidates previous token (detects theft)
- Login always returns same error for wrong email/password (prevents user enumeration)
- Cookie path=/api/auth on refresh_token (limits exposure surface)
- Cookie secure=true in production (HTTPS only)
- Cookie sameSite=strict (CSRF protection)
- JwtAuthGuard registered globally via APP_GUARD — all routes protected by default
- Use @Public() decorator to opt-out specific routes from the global guard

### ITokenUser shape (returned in login/refresh/me):
{
  id: number,
  email: string,
  firstName: string,
  lastName: string,
  role: 'USER' | 'ADMIN' | 'MANAGER',
  active: boolean
}

## Development Notes

- All endpoints use the `/api` prefix
- CORS is enabled globally
- Global validation pipe is active
- All responses follow the `ApiResponse<T>` interface from `@cadena24-wms/shared`
- Auth uses HTTP-only cookies (no Authorization header required)
- All routes are protected by JwtAuthGuard by default (opt-out with @Public())
